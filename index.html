Idea intellj 导入maven依赖包
默认情况下，pom.xml变化时，idea 会提示 Auto-Import for Maven projects，点击下载依赖

开启自动加载
File > Settings > Maven > Importing, there is a checkbox that says “Import Maven projects automatically”.

使用命令行加载
If that doesn’t help, then I would suggest to make a full clean-up and start again:
Close your project window (and IntelliJ) and remove all *.iml files and all .idea folders (there should be one per module)
Run mvn clean install from the command line
Re-import the project into IntelliJ and pay attention when it asks you to enable auto-import

create maven-spring-mvc from archetypes

1 [安装模板仓库到本地仓库]

git clone https://github.com/kolorobot/spring-mvc-quickstart-archetype.git
cd spring-mvc-quickstart-archetype
mvn clean install

2 创建项目

（1）使用命令从仓库模板创建项目
mvn archetype:generate \
-DarchetypeGroupId=com.github.spring-mvc-archetypes \
-DarchetypeArtifactId=spring-mvc-quickstart \
-DarchetypeVersion=1.0.0 \
-DgroupId=my.groupid \
-DartifactId=my-artifactId \
-Dversion=version

（2）使用Idea Intellj从仓库模板创建项目
New Project->Maven->Create From archetypes->add archetypes
填写模板归档类型，项目名，版本（可能会有多个版本，添加时注意不要有空格）
GroupId=com.github.spring-mvc-archetypes
ArtifactId=spring-mvc-quickstart
Version=1.0.0
->Next->
填写项目的归档类型，项目名，版本
GroupId=myarchetype
ArtifactId=spring-mvc-quickstart
Version=1.0.0
->finish （从仓库拷贝相应文件及下载依赖）

运行项目

在pom.xml配置运行容器插件

（1）通过mvn命令行运行

mvn test tomcat7:run
mvn test jetty:run

（2） 通过配置Idea 容器运行
Idea 右上角运行配置->Edit Configration->+ maven ->
添加命令(分别可以使用tomcat 或jetty运行)
tomcat7:run
jetty:run

在浏览器中查看网页
