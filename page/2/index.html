mark as directory（目录右键菜单）

exclude
不包含目录，此时目录被标记为空心红框，该目录下的文件不被检测，右键新建文件时，找不到java class
cancel exclusion

create maven-spring-mvc from archetypes

1 [安装模板仓库到本地仓库]

git clone https://github.com/kolorobot/spring-mvc-quickstart-archetype.git
cd spring-mvc-quickstart-archetype
mvn clean install

2 创建项目

（1）使用命令从仓库模板创建项目
mvn archetype:generate \
-DarchetypeGroupId=com.github.spring-mvc-archetypes \
-DarchetypeArtifactId=spring-mvc-quickstart \
-DarchetypeVersion=1.0.0 \
-DgroupId=my.groupid \
-DartifactId=my-artifactId \
-Dversion=version

（2）使用Idea Intellj从仓库模板创建项目
New Project->Maven->Create From archetypes->add archetypes
填写模板归档类型，项目名，版本（可能会有多个版本，添加时注意不要有空格）
GroupId=com.github.spring-mvc-archetypes
ArtifactId=spring-mvc-quickstart
Version=1.0.0
->Next->
填写项目的归档类型，项目名，版本
GroupId=myarchetype
ArtifactId=spring-mvc-quickstart
Version=1.0.0
->finish （从仓库拷贝相应文件及下载依赖）

运行项目

在pom.xml配置运行容器插件

（1）通过mvn命令行运行

mvn test tomcat7:run
mvn test jetty:run

（2） 通过配置Idea 容器运行
Idea 右上角运行配置->Edit Configration->+ maven ->
添加命令(分别可以使用tomcat 或jetty运行)
tomcat7:run
jetty:run

在浏览器中查看网页

Git Reset

git reset可用于不同的Commit创建分支

修改head位置

–soft

修改head位置并重置index

–mixed（默认）

修改head位置，重置index和工作副本

–hard

重置到某个commit,但是本地修改过的文件不会被重置

–keep

切换到前1个commit

git reset –soft HEAD^

前换到前3个commit,该命令会修改当前分支head，通常先创建并检出分支后再重置head

$ git reset –hard HEAD~3

重置某个文件的索引

git reset – frotz.c

切换到前三个分支，但修改过的文件不会被重置

git reset –keep HEAD~3

Git Checkout

git checkout用于从不同分支或者不同Commit检出文件，Head位置不变

创建并切换分支

$ git checkout -b dev

相当于以下两条命令

创建分支

$ git branch dev

切换分支

$ git checkout dev

-f强制切换,会丢弃本地修改

$git checkout -f dev

检出其中某个文件

git checkout – hello.rb

Git Branch

查看本地分支

$ git branch

查看远程分支

$ git branch -r
/master

查看所有分支

$ git branch -a
master
remotes/origin/master

若要查看各个分支最后一个提交对象的信息

$ git branch -v
iss53 93b412c fix javascript issue

  • master 7a98805 Merge branch ‘iss53’
    testing 782fd34 add scott to the author list in the readmes

查看哪些分支已经合入当前分支

$ git branch –merged
iss53

  • master

删除分支

#-D 强制执行
$git branch -d master

查看尚未合并的工作：

$ git branch –no-merged
testing

创建分支

git branch test_branch

Git Remote

查看当前配置的远程主机名

$ git remote
origin

查看远程主机的网址

$ git remote -v
origin git@github.com:jquery/jquery.git (fetch)
origin git@github.com:jquery/jquery.git (push)

可以查看该主机的详细信息

$ git remote show <主机名>

添加远程主机

$ git remote add <主机名> <网址>

删除远程主机

$ git remote rm <主机名>

远程主机的改名

$ git remote rename <原主机名> <新主机名>
